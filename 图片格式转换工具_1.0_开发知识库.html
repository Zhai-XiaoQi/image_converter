从一次图片整理任务，沉淀成一个可分享的小工具

这个知识库记录了“图片格式转换工具 1.0”的完整开发过程。目标不是只告诉你代码写了什么，而是让不懂编程的人也能理解：需求怎么拆、功能怎么一步步长出来、每个模块用了什么技术、最后怎么打包给别人用。

1. 为什么做这个工具

最开始的需求不是“做软件”，而是一个很具体的素材处理问题：图片按语言和类型整理后，还要处理 PNG、JPG、WEBP、渐进式 JPG 等格式。手动处理可以做一次，但每次都靠人工复制、改名、检查、转换，会有三个问题：

  • 容易出错：图片多、语言多、目录多，手动移动和转换容易漏图、错图。
  • 重复劳动：同类任务以后还会出现，不能每次都从头做。
  • 协作门槛高：如果发给运营或同事，他们不一定会 Python，也不适合让他们跑命令。
核心思路：先把一次具体任务做对，再把可重复的步骤抽出来，变成可视化工具，最后打包成 EXE 给别人双击使用。

2. 开发路线

0. 图片整理任务

先按语言和大类整理素材，确认文件数量、PNG 明细、语言是否放错。这个阶段证明了“文件处理可以交给脚本”。

1. 脚本版格式转换

把 PNG、JPG、WEBP 转换逻辑做成 Python 脚本，支持质量、透明背景、保留目录结构、报告输出。

2. Tkinter 可视化界面

从命令行升级为窗口软件，用户可以选择文件夹、选择输出格式、点击按钮转换，不需要懂命令。

3. 目录树和网格预览

增加左侧文件树、右侧缩略图预览、格式筛选、模糊搜索、勾选处理，减少误操作。

4. 单图编辑能力

增加裁切框、拖拽、吸附、比例预设、撤销/重做、缩放、旋转、对齐和保存副本。

5. 选择同步和体验优化

右侧预览改成只显示左侧勾选的图片，批量页和单图页用 Tab 区分，顶部布局压缩，默认窗口居中。

6. EXE 打包分享

用 PyInstaller 打包成无黑窗 EXE，并生成 zip 分享包。别人不需要安装 Python。

5. 核心结构

这个工具主要由三个对象组成。小白可以把它理解成：一个负责主窗口，一个负责一张待转换图片的数据，一个负责单图编辑。

ImageConverterApp

主程序。负责创建窗口、扫描图片、显示目录树和预览、执行批量转换、打开单图处理页。

ConvertJob

每张图的一条任务记录。里面包含原路径、目标路径、是否选中、树节点 ID、状态和消息。

ImageEditorWindow

弹窗式单图编辑器。处理裁切框、拖拽、缩放、旋转、保存副本、覆盖原图。

EmbeddedImageEditor

嵌入式单图编辑器。复用弹窗编辑逻辑，放到“单图处理”Tab 页面里。

6. 批量转换模块

它做什么

批量转换模块负责把一批图片统一转成 JPG、PNG 或 WEBP。用户可以选择一个文件夹，也可以选择多个独立文件。

关键技术

  • Path.rglob("*")：递归扫描子文件夹。
  • SUPPORTED_INPUTS：限制输入格式，避免把非图片文件当图片处理。
  • Pillow Image.save：按不同格式保存输出。
  • conversion_report.txt：记录成功、失败、跳过、未选数量。

为什么要保留目录结构

素材常常不是平铺在一个文件夹里，而是按语言、渠道、素材类型分层。如果转换后全部丢到一个目录，会失去上下文，还可能重名覆盖。因此工具支持“保留目录结构”。

注意：输出目录不要放在输入目录内部，否则容易把输出结果再次扫进去。工具会尽量跳过当前输出目录，但最稳妥的做法是把输出目录放到输入目录旁边。

7. 目录树与预览模块

预览模块的目标是让用户在转换前看清楚“到底要处理哪些图”。左侧是目录树，右侧是缩略图网格。

左侧目录树

用 ttk.Treeview 展示父文件夹、子文件夹和图片文件。文件夹勾选会联动子项，子项变化也会反映到父文件夹状态。

右侧预览

右侧只显示当前已勾选图片。左侧选什么，右侧就显示什么，避免“看得到但不会处理”的误解。

筛选与搜索

按 JPG、PNG、WEBP、其他格式筛选；搜索框支持按文件名和路径模糊搜索。

性能优化

缩略图按文件路径和修改时间缓存，窗口重排时只重新布局，不重复读取图片。

8. 单图编辑模块

单图编辑是 1.0 里最像“设计工具”的部分。它使用 Tkinter 的 Canvas 实现可交互画布。

核心交互

  • 绿色框代表最终输出区域，拖四边或四角可以改变裁切范围。
  • 鼠标在图片内部时拖动图片，用来调整保留位置。
  • 滚轮缩放图片，旋转按钮可以 90 度旋转。
  • 拖拽边框靠近图片或画布边缘时自动吸附。
  • 比例预设支持 1:1、16:9、9:16、4:3、3:4、3:2。
  • 撤销和重做保存裁切框、图片位置、缩放、旋转等状态。

为什么保存副本

图片编辑有风险，尤其是裁切和覆盖。默认保存副本可以保护原图。副本命名为 原文件名_edited.ext，如果已存在，会自动递增为 _edited_2。

覆盖原图要谨慎：覆盖前工具会弹窗确认，但建议重要素材先保存副本。

9. 打包发布

脚本版需要 Python 和 Pillow，适合开发者自己用。EXE 版适合发给运营、设计或其他同事。

打包技术

使用 PyInstaller。关键参数是 --noconsole，它让软件启动时不弹出黑色命令行窗口。

python -m PyInstaller --noconsole --name 图片格式转换工具 ^
  --distpath image_format_converter_tool\dist ^
  --workpath image_format_converter_tool\build ^
  --specpath image_format_converter_tool ^
  image_format_converter_tool\image_converter_gui.py

分享规则

  • 可以分享：图片格式转换工具_EXE版.zip。
  • 也可以分享整个文件夹：图片格式转换工具_EXE版。
  • 不要只发 图片格式转换工具.exe，因为 _internal 文件夹是运行依赖。
1.0 发布包：EXE 冒烟测试已通过，可以正常启动。zip 大小约 25 MB。

10. 使用说明

批量转换

  1. 打开软件，进入“批量转换”。
  2. 选择文件夹或选择多个文件。
  3. 选择输出格式、质量、输出目录。
  4. 点击“扫描预览”。
  5. 在左侧目录树勾选要处理的图片，右侧会同步显示。
  6. 确认无误后点击“开始转换”。

单图处理

  1. 进入“单图处理”。
  2. 选择图片，或粘贴图片路径。
  3. 拖动绿色框调整裁切区域，拖动图片调整保留位置。
  4. 可使用比例预设、旋转、撤销、重做、对齐等功能。
  5. 点击保存副本，或按目标格式保存转换副本。
  6. 点击“打开结果”查看保存位置。

11. 踩坑与经验

问题 原因 解决方式
双击 BAT 报错 电脑没有识别 Python 命令，或依赖没装。 增加 VBS 无黑窗入口，并最终打包成 EXE。
右侧预览不显示 筛选后卡片没有立即重排，滚动区域没有刷新。 筛选后立即重建卡片并刷新 scrollregion。
子文件夹没读到 早期按文件夹名跳过 converted_images，规则太粗。 改为只跳过当前设置的输出目录本身。
悬停预览抖动 hover 时改变边框宽度，导致布局重排。 hover 只改背景色，不改变卡片尺寸。
EXE 单独发打不开 PyInstaller 文件夹模式需要 _internal。 分享整个文件夹或 zip 包。

12. 可复用模板：如何把重复工作做成工具

第一步：把人工步骤写清楚

不要一开始就说“我要做软件”。先写清楚：输入是什么、输出是什么、中间要做哪些重复动作、哪里最容易出错。

第二步：先做最小脚本

先让脚本能完成核心动作。例如这个项目里，核心动作就是“读取图片，转换格式，保存到输出目录”。

第三步：加可视化

当脚本能跑通后，再加界面，让非技术用户可以选择文件、勾选图片、预览结果。

第四步：加安全保护

包括转换报告、覆盖确认、删除原图确认、默认保存副本等。越是给别人用，越要降低误操作成本。

第五步：打包分享

如果使用者不是开发者，就打成 EXE 或安装包。分享时附上简单说明，告诉对方如何启动、如何避免误操作。

13. 常见问题

渐进式支持全部格式吗？

不支持全部格式。当前工具里的“仅 JPG 渐进式”只对 JPG/JPEG 输出有意义，默认不勾选。

为什么要有脚本版和 EXE 版？

脚本版方便继续开发和修改；EXE 版方便给别人用。二者用途不同，建议都保留。

为什么右侧只显示勾选的图片？

因为右侧预览代表“即将处理的范围”。左侧选什么，右侧显示什么，更不容易误解。

单图编辑会破坏原图吗？

默认保存副本，不破坏原图。只有点击覆盖原图并确认后，才会覆盖。

以后还能继续升级吗？

可以。比如可以做批量尺寸调整、批量水印、更多格式、预设模板、配置保存、安装包等。这些可以作为 1.1 或 2.0。
